# Project Management #18: 구체적으로 요청하기, 모호하게 X

## 원칙
"이 코드를 개선해줘" 대신 "연락처 폼에 이메일 형식과 필수 필드 검증을 추가해줘"

## ❌ 모호한 요청
```
"이 코드를 더 좋게 만들어줘"
"성능을 개선해줘"
"버그를 고쳐줘"
```

## ✅ 구체적 요청
```
"이 SQL 쿼리에 인덱스를 추가하고 N+1 문제를 해결해줘"
"이메일 필드에 정규식 검증과 중복 체크를 추가해줘"
"사용자가 빈 문자열을 입력할 때 발생하는 ValidationError를 수정해줘"
```
