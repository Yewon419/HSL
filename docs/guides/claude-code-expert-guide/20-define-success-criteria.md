# Project Management #20: 완료 기준 명확히 정의

## 원칙
"완료"가 무엇인지 명확히 하세요.

## 예시
```
"로그인 기능을 완료해줘. 완료 기준:
- [ ] 유효한 이메일/비밀번호로 로그인 성공
- [ ] 잘못된 자격증명 시 적절한 오류 메시지
- [ ] JWT 토큰 생성 및 반환
- [ ] Rate limiting (분당 5회)
- [ ] 모든 테스트 통과
- [ ] API 문서 업데이트"
```

## Done의 정의
- 기능 구현 ✓
- 테스트 작성 및 통과 ✓
- 문서 업데이트 ✓
- 코드 리뷰 통과 ✓
