# Claude Code Expert User Guide - Overview

이 가이드는 Claude Code를 가장 효과적으로 사용하는 전문가들의 노하우를 정리한 것입니다.

## 구성

이 가이드는 16개 카테고리, 200개 이상의 실전 팁으로 구성되어 있습니다:

1. **Git Workflow Practices** (01-12번 파일) - Git을 활용한 버전 관리 전략
2. **Project Management** (13-29번 파일) - 프로젝트 관리 및 작업 구조화
3. **Error Prevention** (30-44번 파일) - 오류 방지 전략
4. **Communication Patterns** (45-56번 파일) - 효과적인 소통 방법
5. **File Management** (57-68번 파일) - 파일 관리 모범 사례
6. **Incremental Development** (69-76번 파일) - 점진적 개발 방법
7. **Verification Workflows** (77-88번 파일) - 검증 워크플로우
8. **Recovery Strategies** (89-96번 파일) - 복구 전략
9. **Context Management** (97-111번 파일) - 컨텍스트 관리
10. **Collaboration Tips** (112-124번 파일) - 팀 협업 팁
11. **Advanced Features** (125-148번 파일) - 고급 기능 활용
12. **Performance & Optimization** (149-160번 파일) - 성능 최적화
13. **Security & Code Quality** (161-173번 파일) - 보안 및 코드 품질
14. **Common Anti-Patterns** (174-182번 파일) - 피해야 할 안티패턴
15. **Documentation** (183-192번 파일) - 문서화 모범 사례
16. **Real-World Applications** (193-204번 파일) - 실제 적용 사례

## 핵심 원칙

전문가들이 공통적으로 강조하는 핵심 원칙:

- **명확하고 구체적으로** - 명확한 지시가 더 나은 결과를 낳습니다
- **반복하고 개선하기** - 첫 시도는 좋고, 2-3번 반복하면 훌륭합니다
- **테스트 주도 개발** - 테스트를 먼저 작성하고 Claude가 구현하게 합니다
- **작고 점진적인 변경** - diff를 작게 유지하고 각 단계를 검증합니다
- **컨텍스트 적극 관리** - /clear를 자주 사용하고 CLAUDE.md를 집중적으로 유지합니다
- **파트너로 협업** - 맹목적으로 위임하지 말고 적극적으로 협업합니다
- **검증을 통한 안전성** - 진행하기 전에 테스트, 리뷰, 체크포인트를 만듭니다
- **학습하고 문서화** - 인사이트를 CLAUDE.md에 기록하여 향후 세션에 활용합니다
- **자동화 활용** - 커스텀 명령, MCP 서버, headless 모드를 활용합니다
- **사람이 중심** - 항상 변경사항을 검토하고 승인합니다. 코드 품질에 대한 책임은 사용자에게 있습니다

## 사용 방법

1. 필요한 카테고리의 파일들을 읽어보세요
2. 자신의 워크플로우에 적용 가능한 팁을 선택하세요
3. 작은 것부터 시작해서 점진적으로 개선하세요
4. 효과가 있는 방법을 CLAUDE.md에 문서화하세요

## 시작하기 전에

**가장 중요한 팁:**
- 모든 작업 전에 Git 저장소를 만들고 초기 커밋을 생성하세요
- 큰 변경 전에는 항상 브랜치를 만드세요
- Claude에게 계획을 먼저 물어보세요
- 작은 단위로 작업하고 각 단계를 검증하세요
