## 시스템 개요

RSI+MA 자동매매 시스템은 기술적 지표를 활용하여 매수/매도/손절 시그널을 자동으로 감지하고, 카카오톡/Slack을 통해 실시간 알림을 전송하는 시스템입니다.

### 핵심 기능

- ✅ **자동 시그널 감지**: RSI < 30 AND MA20 > MA50 조건의 매수 시그널 자동 탐지
- ✅ **실시간 알림**: 카카오톡, Slack, 이메일로 시그널 전송
- ✅ **포지션 관리**: 매수/매도 이력 추적 및 수익률 계산
- ✅ **백테스팅**: 과거 데이터 기반 전략 성과 검증
- ✅ **대시보드**: 웹 기반 시그널 모니터링

### 기술 스택

- **Backend**: FastAPI (Python)
- **Database**: PostgreSQL + TimescaleDB
- **Task Queue**: Apache Airflow
- **Frontend**: HTML/CSS/JavaScript
- **Notifications**: KakaoTalk API, Slack Webhooks, SMTP

## 매매 전략

### RSI+MA 상승전략

**핵심 아이디어**: 과매도 구간에서 상승 추세를 보이는 종목을 조기에 포착

#### 매수 조건
```
RSI < 30 (과매도 구간)
AND
MA20 > MA50 (단기 이동평균이 중기 이동평균보다 높음 = 상승 추세)
```

#### 매도 조건
```
RSI > 70 (과매수 구간)
```

#### 손절 조건
```
현재가 <= 매수가 × 0.95 (5% 손실)
```

#### 목표 수익
```
목표가 = 매수가 × 1.10 (10% 수익)
```

### 전략 설명

1. **RSI < 30**: 종목이 과도하게 매도되어 반등 가능성이 높은 상태
2. **MA20 > MA50**: 단기적으로 상승 추세를 유지하고 있는 상태
3. **두 조건 동시 충족**: 상승 추세 내에서 일시적으로 과매도된 종목 = 매수 기회

### 2. 카카오톡 설정

#### Step 1: 카카오 개발자 등록
1. https://developers.kakao.com 접속
2. 로그인 후 "내 애플리케이션" 메뉴
3. "애플리케이션 추가하기"
4. 앱 이름 입력 후 생성

#### Step 2: 카카오 로그인 활성화
1. 생성한 앱 선택
2. "카카오 로그인" 메뉴
3. "활성화 설정" ON
4. Redirect URI 등록: `http://localhost:8000`

#### Step 3: Access Token 발급

**방법 A: REST API 직접 호출**
```bash
# 1. REST API 키 확인 (앱 설정 > 앱 키)
REST_API_KEY="your-rest-api-key"

# 2. 인증 코드 받기 (브라우저에서 접속)
# https://kauth.kakao.com/oauth/authorize?client_id=${REST_API_KEY}&redirect_uri=http://localhost:8000&response_type=code

# 3. 리다이렉트된 URL에서 code 파라미터 복사
# 4. Access Token 발급
curl -X POST "https://kauth.kakao.com/oauth/token" \
  -d "grant_type=authorization_code" \
  -d "client_id=${REST_API_KEY}" \
  -d "redirect_uri=http://localhost:8000" \
  -d "code={복사한_CODE}"
```

**방법 B: 카카오 개발자 도구 사용**
1. https://developers.kakao.com/tool/rest-api/open/get/v2-user-me
2. "테스트" 버튼 클릭
3. 자동 생성된 Access Token 사용

#### Step 4: 웹 UI에서 사용
```
http://localhost:8000/send_notification.html
```
- 채널: 카카오톡 선택
- Access Token 붙여넣기
- "시그널 전송하기" 클릭

| quantity | INTEGER | 수량 |

| stop_loss_price | NUMERIC(10,2) | 손절가 |
| sell_date | TIMESTAMP | 매도일 |
| sell_price | NUMERIC(10,2) | 매도가 |
| profit_loss | NUMERIC(12,2) | 손익 금액 |
| profit_loss_rate | NUMERIC(6,2) | 손익률 (%) |
| status | VARCHAR(20) | 상태 (OPEN/CLOSED) |
| notes | TEXT | 메모 |

### 3. notification_logs (알림 로그)

| 컬럼 | 타입 | 설명 |
|------|------|------|
| id | SERIAL | 로그 ID (PK) |
| signal_id | INTEGER | 시그널 ID (FK) |
| channel | VARCHAR(20) | 채널 (email/kakao/slack) |
| recipient | VARCHAR(100) | 수신자 |
| subject | VARCHAR(200) | 제목 |
| message | TEXT | 메시지 내용 |
| sent_at | TIMESTAMP | 전송 시각 |
| success | BOOLEAN | 성공 여부 |
| error_message | VARCHAR(500) | 에러 메시지 |

### 뷰 (Views)

#### v_open_positions_summary
보유 포지션 + 현재가 + 수익률 계산

#### v_recent_signals
최근 100개 시그널 조회

#### v_trading_performance
매매 성과 통계 (승률, 평균 수익률 등)

## 문제 해결

### 1. Docker 컨테이너 시작 실패

```bash
# 로그 확인
docker-compose logs backend
docker-compose logs postgres

# 컨테이너 재시작
docker-compose restart backend

# 전체 재시작
docker-compose down
docker-compose up -d
```

### 2. 시그널이 조회되지 않음

**원인**: 데이터가 수집되지 않았거나 지표 계산이 안됨

```bash
# 데이터 확인
docker exec -it stock-db psql -U admin -d stocktrading

# SQL 실행
SELECT MAX(date) FROM stock_prices;
SELECT COUNT(*) FROM technical_indicators WHERE date = '2025-10-02';

# 수동 데이터 수집
docker exec stock-backend python collect_missing_data.py

# 지표 계산
docker exec stock-backend python simple_indicators_oct2.py
```

### 3. 알림 전송 실패

**Slack 오류**:
```
"Slack send failed - check webhook URL"
```
→ Webhook URL이 올바른지 확인

**카카오톡 오류**:
```
"KakaoTalk send failed: 401"
```
→ Access Token이 만료됨. 재발급 필요

**일반 오류**:
```bash
# 백엔드 로그 확인
docker logs stock-backend -f

# 에러 메시지 확인 후 해당 설정 수정
```

### 4. None 값 오류

```
TypeError: unsupported format string passed to NoneType.__format__
```

**해결**: 이미 수정 완료됨. 백엔드 재시작
```bash
docker-compose restart backend
```

### 5. 데이터베이스 연결 오류

```bash
# 데이터베이스 상태 확인
docker-compose ps stock-db

# 연결 테스트
docker exec -it stock-db psql -U admin -d stocktrading -c "SELECT 1;"

# 테이블 존재 확인
docker exec -it stock-db psql -U admin -d stocktrading -c "\dt"
```
